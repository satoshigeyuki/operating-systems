アクセス方法

  • Jupyter環境に行き
  • -> assignments
  • -> Fetch feedback
  • -> 最後にsubmitしたものに点数がついています

フィードバックの見方

  • それぞれの問題の結果は右上の score を見てください
  • t1-1-1のようにtで始まるセルではテストが自動的に 行われていて, そのテスト結果はその下にあらわれているが, scoreはそれとは異なる場合がある(例えば想定していた正解が "sbrk" でも "brk" も正解としている場合などがあり, その場合テスト結果ではエラーとなっていても score では ◯になっているはずである. 自動的なテストの結果が採点にそのまま 用いられているわけではない
  • コーディングの問題(4問)は基本的にこちらが用意した テストデータに正解を出すかと, 問題の指示(例えば「排他制御を用いずに」など) に従っているかどうかで見ている. 途中点はつけていない. テストデータは試験中に提供されていたものと, 採点時のみ利用したものがあるため, 試験中のテストデータを通過していても正解となっていない場合もある (間違いの中には, もう少し試験中のテストデータでそれが見つかるように しておいてあげればよかったというものもある)
  • 記述式の問題ではいくつかの採点基準を設けて途中点をつけている.
  • 減点理由をコメントしている場合もある(全てではない).
